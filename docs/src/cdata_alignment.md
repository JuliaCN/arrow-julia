# C Data Interface Alignment

This page tracks the current Arrow.jl alignment with Apache Arrow's C
interfaces. The main implemented surface is the same-process
[Arrow C Data Interface](https://arrow.apache.org/docs/format/CDataInterface.html)
for `ArrowSchema` and `ArrowArray` pointers. Arrow.jl also exposes a bounded
producer for the
[Arrow C Stream Interface](https://arrow.apache.org/docs/format/CStreamInterface.html)
through `ArrowArrayStream` callbacks over those C Data layouts. The
[Arrow C Device data interface](https://arrow.apache.org/docs/format/CDeviceDataInterface.html),
and
[Arrow PyCapsule Interface](https://arrow.apache.org/docs/format/CDataInterface/PyCapsuleInterface.html)
are adjacent specifications and are tracked separately below.

Status labels:

- `Covered`: implemented and covered by focused tests.
- `Partial`: implemented for the listed subset, with a recorded audit item.
- `Not implemented`: a known gap inside the C Data boundary.
- `Out of scope`: an adjacent Arrow C interface or transport surface, not a
  C Data base-struct feature.

## Synchronization Matrix

| Area | Official Arrow contract | Arrow.jl export | Arrow.jl import | Evidence | Status | Next action |
| --- | --- | --- | --- | --- | --- | --- |
| Base struct ABI | `ArrowSchema` and `ArrowArray` carry format, metadata, children, dictionary, buffers, release callbacks, and producer private data. | `Arrow.CData.ArrowSchema`, `Arrow.CData.ArrowArray`, `exporttable`, and `exporttable!`. | `importtable` borrows a top-level struct array and rejects impossible format, length, null-count, buffer-count, child-count, child-pointer, and dictionary-peer shapes through focused import submodules. | `src/cdata/core.jl`, `src/cdata/export.jl`, `src/cdata/import.jl`, `src/cdata/import/`, `test/cdata.jl`. | Covered | Keep ABI field order, modular import validation, and release behavior guarded by focused tests. |
| Lifetime and release callbacks | Consumers release only the base structures; producers release children, dictionaries, and private data. Released structures set `release` to `NULL`. | Export owners retain schema strings, metadata bytes, child refs, pointer arrays, dictionaries, and source buffers. | Imported tables call base `release` callbacks through `release!` and reject released top-level, child, and dictionary schema/array inputs. | C smoke, embedded C smoke, release-idempotency tests, and released-input regression tests. | Covered | Keep C smoke in CI and avoid exposing child-release helpers as public API. |
| Same-process zero-copy | C Data exposes pointers to existing Arrow memory; it is not an IPC, persistence, or cross-process transport. | Exported arrays and C Stream batches point at Arrow.jl-owned buffers. | Imported primitive, bitmap, offset, data, view, list, union, and REE buffers are borrowed with `own=false`; imported C Stream batches reuse the same C Data borrowed-vector path. | `test/cdata_validation_report.jl`, pointer-identity assertions in `test/cdata.jl`. | Covered | Keep reporting warmed base C Data and C Stream export/import/scan timings plus pointer identity, with stream single-batch pull separated from consumer-side collection overhead. |
| C consumer header | The C Data and C Stream specifications are small enough to copy into consumers. | `include/arrow_julia_cdata.h` exposes package-owned struct/release helpers for `ArrowSchema`, `ArrowArray`, and `ArrowArrayStream`. | Consumers can validate and release exported base structs and streams from C. | `header_path`, `test/cdata_smoke.c`, `test/cdata_embed_smoke.c`. | Covered | Keep the header limited to low-level C interface helpers, not a high-level table builder. |
| Schema and field metadata bytes | `ArrowSchema.metadata` uses the official native-endian key/value byte layout; absent metadata can be `NULL`. Extension type names and extension metadata are encoded there. | Schema and field metadata are exported from `Arrow.getmetadata` and `Arrow.withmetadata`. | Imported tables and columns expose raw schema/field metadata through `Arrow.getmetadata`, including `ARROW:extension:*` keys as a storage-preserving fallback, and reject negative entry counts or string byte lengths. | Metadata export/import tests, malformed metadata tests, and `ImportedTable` metadata fields. | Covered | None. |
| Null layout | Format `n` represents null arrays with no buffers. | `NullVector` exports as `n`. | Format `n` imports as a missing-valued vector. | Null/logical scalar tests. | Covered | None. |
| Primitive integers and floats | Fixed-width primitive formats use one optional validity bitmap and one data buffer. | Primitive storage exports standard format strings. | Imported columns borrow the data buffer, with nullable wrappers when needed. | Primitive and nullable primitive tests. | Covered | None. |
| Boolean | Format `b` uses bit-packed boolean values plus optional validity. | `BoolVector` exports as `b`. | Imported boolean columns borrow packed data and validity bitmaps. | Boolean import tests. | Covered | None. |
| UTF-8 and binary | Formats `u`, `U`, `z`, and `Z` use offsets plus data buffers. | String, large string, binary, and large binary Arrow columns export standard layouts. | Imported columns validate offsets, validate UTF-8 bytes for string columns, then borrow offsets and data buffers. | UTF-8, large UTF-8, binary, large binary, malformed-offset, and malformed-UTF-8 tests. | Covered | None. |
| UTF-8 view and binary view | Formats `vu` and `vz` use view records, variadic data buffers, and a variadic-buffer-lengths buffer in C Data. | Arrow.jl view columns export `vu` and `vz` with the extra C Data lengths buffer. | Imported view columns validate spans, validate UTF-8 bytes for UTF-8 view columns, and borrow inline, view, data, and lengths storage. | View export/import tests plus malformed UTF-8 view tests. | Covered | None. |
| List and large list | Formats `+l` and `+L` use offsets and one child array. | List columns export normal and large offset widths. | Imported list columns borrow offsets and child arrays. | List and large-list tests. | Covered | None. |
| List-view and large list-view | Formats `+vl` and `+vL` use offsets, sizes, and one child array. | `Arrow.ListView` columns export normal and large offset widths. | Imported list-view columns borrow offsets, sizes, and child arrays. | List-view export/import tests. | Covered | None. |
| Fixed-size binary and fixed-size list | Formats `w:N` and `+w:N` use fixed-width data or one fixed-size-list child. | Fixed-size binary/list columns export standard layouts. | Imported columns borrow data or child arrays. | Fixed-size tests. | Covered | None. |
| Struct | Format `+s` uses one validity buffer and child arrays. | Top-level tables and nested struct columns export as struct arrays. | Top-level structs and nested struct columns import; nullable top-level rows wrap imported columns so invalid rows read as `missing`. | Struct tests and top-level validation tests. | Covered | None. |
| Map | Format `+m` uses Int32 offsets and one entries struct child. | Standard Int32-offset map columns export. Large-offset Arrow.jl map views are rejected because C Data map format has no offset-width variant. | Imported map columns borrow Int32 offsets and entries. | Map tests and explicit large-offset rejection. | Covered | None. |
| Dictionary | Parent schema/array describe indices; dictionary schema/array describe values. | Dictionary columns export parent indices plus dictionary values, including nested dictionary value arrays when supplied by Arrow.jl vectors. | Imported dictionary columns borrow parent indices, nested dictionary indices, and final dictionary value buffers recursively; non-null indices are validated against the dictionary length. | Dictionary, nested-dictionary, and malformed-index import tests. | Covered | None. |
| Dense and sparse union | Formats `+ud:<ids>` and `+us:<ids>` carry type IDs and optional dense offsets. | Dense and sparse union columns export declared UInt8 type IDs. | Imported union columns borrow type IDs, dense offsets, and child arrays. | Union tests. | Covered | None. |
| Run-end encoded | Format `+r` uses `run_ends` and `values` children. | Run-end encoded columns export `+r`. | Imported REE columns borrow both child arrays and validate run-end shape. | REE import/export and malformed run-end tests. | Covered | None. |
| Logical scalar storage | Decimal, date, time, timestamp, duration, and interval storage use C Data format strings over physical buffers. | Arrow.jl-surfaced decimal, date, time, timestamp, duration, year-month interval, day-time interval, and month-day-nano interval storage exports. | The same surfaced logical scalar formats import over borrowed physical buffers. | Logical scalar tests, the format-mapping guard in `test/cdata/base_contract_tests.jl`, and `src/cdata/formats.jl`. | Covered | None. |
| Array offsets | Consumers may receive arrays with non-zero `ArrowArray.offset` when buffers are large enough for `length + offset`. | Export emits zero-offset arrays. | Import honors non-zero offsets for supported top-level and child arrays, including bit-level validity and boolean buffers. | Offset-aware import tests in `test/cdata.jl`. | Covered | None. |
| C Stream Interface | `ArrowArrayStream` is a pull-style stream of schemas and arrays using callbacks. | `exportstream` and `exportstream!` export an `Arrow.Table` as one pull-style `ArrowArrayStream` batch, followed by a released end-of-stream array; the single-batch producer precomputes schema/array exports and transfers cached ownership through the callbacks. | `importstream` borrows an `ArrowArrayStream` as an iterator of release-governed `ImportedStreamBatch` table views. | `src/cdata/stream.jl`, `include/arrow_julia_cdata.h`, `test/cdata/stream_tests.jl`, `test/cdata_smoke.c`. | Covered | Keep stream producer/import release behavior and unconsumed cached-export cleanup guarded by focused tests. |
| C Device data interface | Device arrays and device streams carry memory-manager and device placement information. | Not exported as C Device. | Not imported as C Device. | Out-of-scope docs boundary. | Out of scope | `CDATA-AUDIT-009`. |
| PyCapsule Interface | Python objects can expose Arrow C Data, Stream, and Device interfaces through capsule methods. | No PyCapsule producer. | No PyCapsule consumer. | Out-of-scope docs boundary. | Out of scope | `CDATA-AUDIT-010`. |

## Audit Tracker

| ID | Gap or risk | Evidence | Owner surface | Priority | Tracking status | Landing condition |
| --- | --- | --- | --- | --- | --- | --- |
| CDATA-AUDIT-001 | Imported C Data metadata is borrowed but not exposed through a high-level `Arrow.getmetadata` surface on `ImportedTable` or imported columns. | Imported tables now decode the C Data metadata byte layout and metadata-bearing imported columns use a delegating wrapper that preserves low-level buffer access. | `src/cdata/core.jl`, `src/cdata/imported_vectors.jl`, `src/cdata/import.jl`, `test/cdata/import_basic_tests.jl`. | P1 | Closed | Landed when imported schema and field metadata became readable without copying data buffers, with tests for schema and column metadata. |
| CDATA-AUDIT-002 | Extension type reconstruction from C Data metadata is not implemented. | C Data import now preserves `ARROW:extension:name` and `ARROW:extension:metadata` on imported storage vectors as a documented raw fallback; it does not reconstruct Julia extension wrapper values. | `src/cdata/imported_vectors.jl`, `src/cdata/import.jl`, `test/cdata/import_basic_tests.jl`, `docs/src/manual.md`. | P1 | Closed | Landed when C Data import recognized extension metadata through `Arrow.getmetadata` and documented raw storage fallback behavior. |
| CDATA-AUDIT-003 | List-view export had no native Arrow.jl producer vector shape. | `Arrow.ListView` now carries offsets, sizes, and child values; `exporttable` emits `+vl` / `+vL` and tests verify borrowed validity, offsets, sizes, and child buffers. | `src/arraytypes/list.jl`, `src/cdata/export.jl`, `test/cdata/export_tests.jl`. | P2 | Closed | Landed when a native producer shape could emit `+vl` / `+vL` with pointer-identity tests. |
| CDATA-AUDIT-004 | Nullable top-level struct import is rejected. | `importtable` now accepts top-level struct validity and wraps each imported column with a row-validity vector while preserving child buffer borrowing. | `src/cdata/import.jl`, `src/cdata/imported_vectors.jl`, `test/cdata/import_nested_tests.jl`. | P2 | Closed | Landed when nullable top-level rows returned `missing` through imported columns without copying child buffers. |
| CDATA-AUDIT-005 | Nested dictionary chains were rejected on import. | `_import_dictionary_column` now imports dictionary values recursively, and nested-dictionary tests verify parent indices, nested indices, and final string buffers remain borrowed. | `src/cdata/import.jl`, imported vector wrappers, `test/cdata/import_basic_tests.jl`. | P2 | Closed | Landed when nested dictionary values imported correctly with targeted zero-copy pointer checks. |
| CDATA-AUDIT-006 | Month-day-nano interval (`tin`) was not represented in the C Data logical scalar mapping. | Arrow.jl now exposes `Meta.IntervalUnit.MONTH_DAY_NANO` with `MonthDayNanoInterval` storage, and C Data export/import maps it to `tin` over borrowed 16-byte interval buffers. | `src/metadata/Schema.jl`, `src/eltypes.jl`, `src/cdata/formats.jl`, `test/cdata/import_nested_tests.jl`. | P2 | Closed | Landed when `tin` exported/imported with a focused pointer-identity test. |
| CDATA-AUDIT-007 | Non-zero `ArrowArray.offset` import was rejected. | Offset-aware import tests now cover top-level struct slicing plus primitive, boolean, UTF-8, list, dictionary, fixed-size, view, and run-end encoded child arrays. | `src/cdata/import.jl` and imported vector wrappers. | P1 | Closed | Landed when import borrowed non-zero-offset arrays without misaddressing validity, offsets, values, or child buffers. |
| CDATA-AUDIT-008 | C Stream was not part of the package-owned C interface surface. | `Arrow.CData.ArrowArrayStream`, `exportstream`, `exportstream!`, and `importstream` now cover same-process C Stream producer and consumer behavior over the existing C Data exporter/importer, with C smoke coverage for schema, batch, end-of-stream, release behavior, cached schema/array transfer, and unconsumed cached-export cleanup. | `src/cdata/stream.jl`, `include/arrow_julia_cdata.h`, `test/cdata/stream_tests.jl`, `test/cdata_smoke.c`. | P2 | Closed | Landed when Arrow.jl could export and import `ArrowArrayStream` batches with release-governed borrowed table views. |
| CDATA-AUDIT-009 | C Device data is not part of `Arrow.CData`. | Device arrays add device and memory-manager semantics beyond host buffers. | Future device-interface module or explicit non-goal. | P3 | Deferred | A separate design decides whether Arrow.jl should expose device arrays/streams. |
| CDATA-AUDIT-010 | PyCapsule producer/consumer methods are not implemented. | Arrow.jl currently exposes Julia FFI pointers, not Python object capsule protocols. | Python interop boundary, if adopted. | P3 | Deferred | A package-owned interop design defines capsule ownership, release behavior, and tests against Python consumers. |
| CDATA-AUDIT-011 | Performance governance was a local receipt, not a CI gate. | `test/cdata_validation_report.jl` now warms C Data/C Stream hot paths before steady-state measurement, reports base export/import/scan timings, separates C Stream producer setup, open, single-batch import, collection, and scan timings, validates pointer identity on both import paths, and CI gates steady-state base plus stream export/import allocation limits while leaving wall-clock timing informational by default. ## Follow-Up Order

1. Keep C Data and C Stream release behavior covered by focused C and Julia
   tests.
2. Keep `CDATA-AUDIT-009` and `CDATA-AUDIT-010` outside the current C Data
   base-scope unless a separate device or Python interop design is accepted.
